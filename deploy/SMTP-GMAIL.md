# Настройка Gmail SMTP для DierCHAT

Заполнять нужно **на сервере** в файле `/opt/dierchat/config.json` (или скопировать сюда значения из этого шаблона перед деплоем).

## 1. Пароль приложения Google (не обычный пароль почты)

1. Войдите в аккаунт Google → [Безопасность](https://myaccount.google.com/security).
2. Включите **двухэтапную аутентификацию** (если ещё не включена).
3. В разделе «Вход в аккаунт Google» откройте **Пароли приложений** (или [прямая ссылка](https://myaccount.google.com/apppasswords)).
4. Создайте пароль для приложения (тип «Почта», устройство «Другое» — например `DierCHAT`).
5. Скопируйте **16 символов** (без пробелов) — это значение для поля **`password`** в JSON.

## 2. Блок `smtp` для `config.json`

Подставьте **один и тот же** адрес почты в `login` и `from`, и **пароль приложения** в `password`:

```json
    "smtp": {
        "host": "smtp.gmail.com",
        "port": 587,
        "login": "ваш_ящик@gmail.com",
        "password": "xxxxxxxxxxxxxxxx",
        "from": "ваш_ящик@gmail.com"
    }
```

- `login` — полный адрес Gmail, от имени которого идёт отправка.
- `password` — только **пароль приложения** (16 символов).
- `from` — обычно тот же адрес, что и `login`.

## 3. На сервере после правки

```bash
nano /opt/dierchat/config.json
systemctl restart dierchat
journalctl -u dierchat -n 30 --no-pager
```

При успешной отправке в логе будет строка вида `[Email] sent to ...`.  
Если снова видите только `[AUTH] Verification code ...` — проверьте, что в JSON нет опечаток и что `host`/`from`/`login`/`password` не пустые.

## 4. Если Gmail не подходит

Можно использовать SMTP вашего домена (REG.RU, Yandex 360, Mail.ru для бизнеса и т.д.) — уточните у провайдера **хост**, **порт** (часто 587 или 465) и способ авторизации.

# DIERbrowser — план и уточнения (ТЗ §44–45)

## Название продукта

Официальное имя отдельного клиента: **DIERbrowser** (не «Dier Chat Browser»).  
Сборки: `DIERbrowser.exe` (Windows), `DIERbrowser.apk` (Android), опционально отдельный пакет в установщике мессенджера.

## Движок: чтобы «всё работало» (без чёрных iframe)

Рекомендуемая линия, совместимая с требованием полноценного веба:

1. **База: Chromium (C++)** — тот же класс движка, что в Chrome/Edge/Electron `<webview>`.  
   - Полноценные вкладки, DevTools, история, закладки, загрузки, расширения WebExtensions — это **месяцы/годы** инженерии при сборке из исходников Chromium либо интеграции через **CEF** / **WebView2** / готовый **Firefox** (отдельный продукт).

2. **Вместо Python в конвейере сборки** (как в `firefox-main` / `mach`): для своих скриптов автоматизации использовать **C# (.NET)** — кроссплатформенные CLI-утилиты, вызов `msbuild`/`ninja` через `Process`, публикация артефактов, подпись, упаковка MSI/APK.  
   - Python в upstream Firefox остаётся **их** инфраструктурой; наш репозиторий может оборачивать шаги вызовом из **C# host** или **PowerShell**, но целевой «второй язык» плана — **C#**.

3. **Папка исходников браузера** в корне репозитория: **`firefox-main`** (как у вас) — хранится как **reference / будущая интеграция**; отдельный продукт **DIERbrowser** на первом этапе может быть:
   - **Electron + BrowserWindow + webview** с полноценным Chrome внутри, **или**
   - **.NET + WebView2** (Windows) / **CEF** (кроссплатформа).

4. **Связка с DierCHAT (опционально)**: для обычного веб-сёрфинга **подключение к DierCHAT-Server не требуется**. Если нужен «один аккаунт» с мессенджером — отдельно: токен при запуске (`dierchat://auth?token=…`), IPC, своя страница на вашем хосте — это не часть движка и не обязательно для работы браузера.

## Мессенджер vs браузер

| Клиент | Содержимое |
|--------|------------|
| **DierCHAT** (мессенджер) | Чаты + лёгкий встроенный браузер (текущий `BrowserPanel` / webview). |
| **DIERbrowser** | Отдельное приложение с полноценным движком; вызывается из мессенджера кнопкой «Открыть в DIERbrowser». |

## Текущая реализация в репозитории

- Папка **`DIERbrowser/`** — клиент **.NET 8 + WebView2**: вкладки, закладки, история, навигация (см. `DIERbrowser/README.md`). Почему не собираем Firefox/Chromium из исходников — `docs/DIERbrowser_ENGINES.md`.
- IPC Electron: `dierchat:open-dier-browser` — запускает `DIERbrowser.exe` рядом с мессенджером или по `DIERBROWSER_PATH`, иначе открывает URL в системном браузере.
